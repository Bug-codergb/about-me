# 前端性能优化的点
## **网络加载优化**
  - 内联小资源（Base64编码的图片或关键CSS）
  - 启用Gzip压缩（服务器配置）
  - 启用cdn加速
  - 缓存策略 （强缓存，协商缓存）
  - 使用http2.0
    - 二进制格式在协议的解析和优化扩展上带来更多的优势和可能
    - 头部压缩，可以极大减少头部开销
    - 户端和服务器可以将 HTTP消息分解为互不依赖的帧，然后交错发送
## **渲染性能优化**
  - JS使用defer或async避免阻塞DOM解析
  - 减少重排（Reflow）与重绘（Repaint）
  - 图片懒加载
  - WebP 替代 PNG/JPG
    - 更小的体积
    - 无损压缩效率更高
    - 支持透明通道
## **JavaScript优化**
  - 代码分割与懒加载
  - Web Worker处理复杂计算
  - 路由级动态加载
